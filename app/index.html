Unit Testing

Awesome since 1999

1999: Test Driven Development

Kent Beck in Extreme Programming

Pros

  1. You'll write tests!
  2. Recognize difficulties early
  3. Less (or no more) regression. (Easier maintenance)
  4. When familiarized, you will take less time to develop

Cons

  1. Takes a bit of time to set up
  2. Might be mechanical
  3. Not easy to know what and how far to test

2003: Behavior Driven Development

Dan North in Introducing BDD

It guides you on what to test

Now you only have the excuse of the little time it takes yo set up...

How can I automate a test in a language that is interpreted by the browser?

Test framework

Most famous are:

  • Jasmine
  • Mocha

Jasmine

  • BDD
  • Spec Like
  • Offers practically everything you need out of the box (mock, asynchrone, ...)

describe('Javascript', function() {
  it('should add two numbers', function() {
    expect(1 + 1).toBe(2);
  });
  it('should work well with booleans', function() {
    expect(true).toBeTruthy();
    expect(true).not.toBeFalsy();
  });
});

Mocha

  • BDD
  • xUnit Like (extensible)
  • Uses add ons
    • 
var assert = require("assert");

describe('Javascript', function() {
  it('should add two numbers', function() {
    assert.equal(1 + 1, 2);
  });
  it('should work well with booleans', function() {
    assert.equal(true, true);
    assert.notEqual(true, false);
  });
});

Karma Test-Runner

  • Manages you tests
  • Autowatches (or not)
  • Integrates well with Grunt
  • Tests in actual Browsers

Selenium's WebdriverJS

Javascript's way of using Selenium


var webdriver = require('selenium-webdriver');

var driver = new webdriver.Builder().
   withCapabilities(webdriver.Capabilities.chrome()).
   build();

driver.get('http://www.google.com');
driver.findElement(webdriver.By.name('q')).sendKeys('webdriver');
driver.findElement(webdriver.By.name('btnG')).click();
driver.wait(function() {
 return driver.getTitle().then(function(title) {
   return title === 'webdriver - Google Search';
 });
}, 1000);

driver.quit();

Protactor for AngularJs

Wrapper around WebdriverJS, providing with angular tools

  • Provides 3 variables: browser, by and element
  • Uses Jasmin (or Mocha) as test framework
  • Friendlier (I find) than WebdriverJS
  • Requires a standalone WebdriverJS server (installed by protractor)

describe('angularjs homepage', function() {
  it('should greet the named user', function() {
    browser.get('http://www.angularjs.org');

    element(by.model('yourName')).sendKeys('Julie');

    var greeting = element(by.binding('yourName'));

    expect(greeting.getText()).toEqual('Hello Julie!');
  });
});

Conclusion

You don't have any excuse left not to test your Javascript (or software in general):

  • It'll help you to develop in a more structured way
  • It'll make you a better developper
  • It'll make you a faster devlopper (eventually!)
  • It'll reduce the overall price of the development: the later the problems are found, the more expansive they are
